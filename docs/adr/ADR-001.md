# ADR-0001: Capstone Framing — Income Tax Assistant

- **Status:** Draft v1
- **Date:** 2026-07-15
- **Author:** Saurabh Mishra

## Context: 
Help Income-tax payers understand complex income tax rules, determine applicable deductions, explain ITR forms, and answer tax related questions.This is required now as the rules are complex and to avod getting extra tax payment notices

## Decision — Solution Framing Canvas

| Box | Your answer |
|-----|-------------|
| **Inputs** | Natural Language questions, Annual Income, Salary Breakup, Investments, Tax regime preference |
| **Outputs** | Clear explanations, Citation to relevant IT rules/acts, tax calculations, benefit of one regime over other, recommended ITR form and why, List of applicable deductions with limits, |
| **Tools** | OpenAI LLM, RAG Search, Vector DAtabase, Document retrieval tool, Calculator, Comparison tool, DB to store audit logs |
| **Memory** | Conversation memories over the last 10-20 questions; Session memory to remember the user-provided financial details. |
| **Autonomy level**| Invoke tools required like calculator, comparison tables, retrieval; ask clarification questions; give recommendations, |
| **Decision boundaries** | 1. Allowed: retrieve docs, search docs, relevant sections of IT act, calculations, comparisons, recommendations with supporting justification. 2. Not Allowed: Submitting tax returns, using user information outside the system, providing recommendations beyond defined limits, making changes to user records.3. Human intervention; on conflicting regulations, tax filing, financial decisions like investments, property purchases.|
|## Consequences |- **Positive:** 1. Saves user's time, 2. Clarifies ambiguities related to taxes, 3. better decision making, 4. transparency  - **Negative / risks:** 1. Hallucination 2. Outdated Information 3. Misinterpretation of legal language
    
- **Things we'll re-visit:** 
    1. Tools required
    2. Decision boundaries|
